About Frameswork

Frame(s)work was born out of the recognition that modern software development is sluggish and costly. As a senior software developer and architect, I saw the introduction of frameworks for web application development creating complexity where it is not required, and server programming using so many libraries and frameworks that the most basic activities are a complicated endeavour. All for what?

On a side note... The dependency on so many frameworks starts with the slippery slope... Why write a solution myself when someone has already created a library? This may sound like a great idea but what developers don't understand is that they just created a dependency they will carry through the entire life of the application. This means tracking any security vulnerabilites being aware of any compatibility issues caused by upgrades.

Frame(s)work provides a simplified methodology to develop & deliver modern applications. It provides a unique user interface design for an experience more akin to a Computer/Laptop and a simplified server communication/data exchange model. Frame(s)work is not meant to be another framework trying to abstract developers from the browser or forcing developers into numerous dependencies.
